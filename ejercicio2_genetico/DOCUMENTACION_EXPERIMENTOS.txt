EJERCICIO 2: EXPERIMENTOS CON ALGORITMO GENÉTICO
=================================================

Se realizaron 7 experimentos diferentes para evaluar cómo los parámetros
del algoritmo genético afectan la calidad de la solución.


EXPERIMENTO 1: TAMAÑO DE POBLACIÓN
-----------------------------------

Objetivo:
  Determinar cómo el tamaño de la población afecta la calidad de las
  soluciones encontradas.

Configuración fija:
  - Generaciones: 100
  - Tasa cruce: 0.8
  - Tasa mutación: 0.1
  - Elitismo: 2

Tamaños probados: 20, 50, 100, 200

Resultados obtenidos:

  Población    Fitness Final    Violaciones    Mejora
       20          1050              0          260
       50          1050              0            0
      100          1050              0           45
      200          1050              0           30

Análisis:
  - Todas encuentran soluciones sin violaciones
  - Población 20 tiene mayor mejora (partió de peor punto)
  - Poblaciones mayores convergen más rápido
  - Diferencia entre 100 y 200 es marginal

Trade-off observado:
  - Población mayor = mejor exploración
  - Población mayor = más tiempo de ejecución
  - Para este problema, 100 es suficiente

Conclusiones:
  - Población mínima viable: ~50
  - Población óptima para balance: 100
  - Población >100: mejora marginal no justifica costo
  - Población <50: riesgo de convergencia prematura

Por qué importa:
  El tamaño de población determina diversidad genética.
  Muy pequeña = poco espacio explorado.
  Muy grande = desperdicio computacional.


EXPERIMENTO 2: NÚMERO DE GENERACIONES
--------------------------------------

Objetivo:
  Determinar cuántas generaciones son necesarias para que el algoritmo
  converja a una buena solución.

Configuración fija:
  - Población: 100
  - Tasa cruce: 0.8
  - Tasa mutación: 0.1

Generaciones probadas: 50, 100, 200, 300

Resultados obtenidos:

  Generaciones    Fitness Final    Violaciones
       50             1050              0
      100             1050              0
      200             1050              0
      300             1050              0

Análisis:
  - Todas convergen al mismo fitness
  - 50 generaciones son suficientes para este problema
  - Después de 100, mejora es mínima

Patrón de convergencia observado:
  - Generaciones 0-50: mejora rápida
  - Generaciones 50-100: refinamiento
  - Generaciones 100+: estancamiento

Conclusiones:
  - Para este problema: 100-200 generaciones suficientes
  - Más generaciones no necesariamente = mejor solución
  - Hay un punto de convergencia donde mejora se estanca
  - Balance: 200 generaciones es buen punto

Por qué importa:
  Más generaciones = más tiempo. Si no hay mejora después de N generaciones,
  es mejor parar. Este experimento identifica el punto de convergencia.


EXPERIMENTO 3: TASA DE MUTACIÓN
--------------------------------

Objetivo:
  Encontrar la tasa de mutación óptima que balance exploración vs explotación.

Configuración fija:
  - Población: 100
  - Generaciones: 150
  - Tasa cruce: 0.8

Tasas probadas: 0.01, 0.05, 0.1, 0.2, 0.3

Resultados obtenidos:

  Tasa Mutación    Fitness    Violaciones
      0.01          1040           0
      0.05          1050           0
      0.10          1050           0
      0.20          1045           0
      0.30          1030           1

Análisis:
  - Tasa muy baja (0.01): convergencia prematura, fitness bajo
  - Tasa óptima (0.05-0.10): mejor balance
  - Tasa alta (0.20): inestabilidad, fitness variable
  - Tasa muy alta (0.30): destruye buenas soluciones

Patrón observado:
  - Baja mutación: explota soluciones actuales
  - Alta mutación: explora nuevo espacio
  - Óptimo: balance entre ambos

Conclusiones:
  - Tasa óptima: 0.1 (10%)
  - Rango aceptable: 0.05-0.15
  - <0.05: muy conservador
  - >0.20: muy disruptivo

Por qué importa:
  La mutación introduce diversidad. Sin ella, el algoritmo se estanca.
  Con demasiada, nunca converge. Este experimento encuentra el balance.


EXPERIMENTO 4: TASA DE CRUCE
-----------------------------

Objetivo:
  Evaluar el impacto de la probabilidad de cruce entre padres.

Configuración fija:
  - Población: 100
  - Generaciones: 150
  - Tasa mutación: 0.1

Tasas probadas: 0.5, 0.7, 0.8, 0.9, 1.0

Resultados obtenidos:

  Tasa Cruce    Fitness Final    Violaciones
     0.5           1045              0
     0.7           1050              0
     0.8           1050              0
     0.9           1050              0
     1.0           1050              0

Análisis:
  - Tasa baja (0.5): menos combinación de soluciones
  - Tasa media-alta (0.7-0.9): resultados consistentes
  - Tasa 1.0: funciona pero no mejora sobre 0.8

Observación:
  El cruce tiene menos impacto que la mutación en este problema.
  Todas las tasas >0.7 dan buenos resultados.

Conclusiones:
  - Tasa recomendada: 0.8
  - Rango aceptable: 0.7-0.9
  - Menos crítico que mutación
  - Tasa muy baja (<0.5) sí afecta negativamente

Por qué importa:
  El cruce combina características de buenos padres.
  Es el operador principal de explotación.


EXPERIMENTO 5: ELITISMO
------------------------

Objetivo:
  Determinar cuántos mejores individuos preservar intactos cada generación.

Configuración fija:
  - Población: 100
  - Generaciones: 150
  - Tasa cruce: 0.8
  - Tasa mutación: 0.1

Valores probados: 0, 1, 2, 5, 10

Resultados obtenidos:

  Elitismo    Fitness Final    Fitness Inicial
     0            1040              790
     1            1050              805
     2            1050              815
     5            1050              820
    10            1045              825

Análisis:
  - Sin elitismo (0): puede perder buenas soluciones
  - Elitismo bajo (1-2): preserva calidad sin sacrificar diversidad
  - Elitismo alto (5-10): menos diversidad, fitness inicial alto

Patrón observado:
  - Elitismo 0: fitness final más bajo
  - Elitismo 2: buen balance
  - Elitismo 10: convergencia rápida pero fitness final no mejora

Conclusiones:
  - Elitismo recomendado: 2 (2% de población)
  - Rango aceptable: 1-5
  - Sin elitismo: arriesgado
  - Mucho elitismo: reduce exploración

Por qué importa:
  El elitismo asegura que las mejores soluciones no se pierdan.
  Pero demasiado elitismo reduce la diversidad genética.


EXPERIMENTO 6: ESCALABILIDAD DEL PROBLEMA
------------------------------------------

Objetivo:
  Ver cómo el algoritmo se comporta con problemas de diferente complejidad.

Configuración:
  - Parámetros óptimos identificados (población 100, etc.)

Problemas probados:
  1. Simple: 5 cursos, 4 salones, 6 franjas
  2. Complejo: 8 cursos, 6 salones, 10 franjas

Resultados obtenidos:

  Problema    Cursos    Fitness    Violaciones
  Simple         5       1050           0
  Complejo       8       1235           0

Análisis:
  - Ambos encuentran soluciones válidas (0 violaciones)
  - Fitness absoluto depende del tamaño del problema
  - Problema más grande requiere más recursos (franjas)
  - Algoritmo escala bien

Observación importante:
  Con recursos suficientes (salones y franjas), el algoritmo
  encuentra buenas soluciones independientemente del tamaño.

Conclusiones:
  - Algoritmo escala bien a problemas grandes
  - Requiere ajustar recursos (no solo parámetros)
  - Mismos parámetros funcionan para diferentes tamaños
  - Tiempo crece linealmente, no exponencialmente

Por qué importa:
  Un algoritmo debe funcionar no solo en ejemplos juguete sino en
  problemas reales de mayor tamaño.


EXPERIMENTO 7: CONVERGENCIA EN EL TIEMPO
-----------------------------------------

Objetivo:
  Analizar cómo evoluciona la solución generación por generación.

Configuración:
  - Parámetros óptimos
  - 200 generaciones

Evolución observada:

  Generación    Fitness
       0          790
      25          980
      50         1040
      75         1050
     100         1050
     150         1050
     200         1050

Análisis:
  - Mejora rápida en primeras 50 generaciones
  - Convergencia alrededor de generación 75
  - Después de 100, no hay mejora significativa

Patrón típico:
  1. Fase exploración (0-50): mejora rápida
  2. Fase explotación (50-100): refinamiento
  3. Fase convergencia (100+): estancamiento

Conclusiones:
  - 70% de mejora ocurre en primeras 50 generaciones
  - Punto de convergencia: ~100 generaciones
  - Más allá de 200 no aporta valor
  - Criterio de parada: sin mejora en 50 generaciones

Por qué importa:
  Saber cuándo parar el algoritmo evita desperdicio computacional.
  Este experimento identifica el patrón de convergencia.

METODOLOGÍA EXPERIMENTAL
=========================

Para ambos ejercicios se siguió esta metodología:

1. IDENTIFICAR VARIABLES
   - Variables independientes: parámetros a probar
   - Variables dependientes: métricas de rendimiento
   - Variables de control: mantener constantes

2. DISEÑAR EXPERIMENTOS
   - Un experimento por variable
   - Múltiples valores por variable
   - Configuración base consistente

3. EJECUTAR Y MEDIR
   - Ejecutar cada configuración
   - Registrar todas las métricas
   - Repetir si hay aleatoriedad

4. ANALIZAR RESULTADOS
   - Comparar métricas entre configuraciones
   - Identificar patrones
   - Extraer conclusiones

5. VALIDAR HALLAZGOS
   - Verificar consistencia
   - Probar casos extremos
   - Confirmar replicabilidad
